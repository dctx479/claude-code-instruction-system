启动Agent群体模式处理大规模任务: $ARGUMENTS

## 群体模式适用场景
- 大规模代码迁移
- 全项目重构
- 批量文件处理
- 广度优先的探索任务

## 执行流程

### 1. 任务分解
将大任务分解为大量小任务:
```
大任务 → [小任务1, 小任务2, ..., 小任务N]
```

### 2. 任务队列管理
```
创建任务队列:
- 待执行队列
- 执行中队列 (最大并行数)
- 已完成队列
- 失败队列 (待重试)
```

### 3. Worker分配
```
根据任务类型分配Worker:
- 简单任务 → haiku模型 (快速、低成本)
- 复杂任务 → sonnet模型 (能力、成本平衡)
```

### 4. 进度追踪
```
实时显示:
- 总任务数
- 已完成数
- 进行中数
- 失败数
- 预估剩余时间
```

### 5. 结果聚合
```
收集所有Worker输出:
- 合并成功结果
- 记录失败任务
- 生成汇总报告
```

## 并行度配置

| 任务规模 | 推荐并行度 | 模型选择 |
|----------|------------|----------|
| < 10 | 3-5 | sonnet |
| 10-50 | 5-8 | haiku为主 |
| 50-100 | 8-10 | haiku |
| > 100 | 10+ | haiku |

## 输出格式

```markdown
## 群体执行报告

### 任务概览
- 总任务数: N
- 成功: X (Y%)
- 失败: Z
- 跳过: W

### 进度
[=========>        ] 60% (30/50)

### 执行详情
| 批次 | 任务范围 | 成功 | 失败 | 耗时 |
|------|----------|------|------|------|
| 1 | 1-10 | 10 | 0 | 30s |
| 2 | 11-20 | 9 | 1 | 35s |
| ... | ... | ... | ... | ... |

### 失败任务
[失败任务列表和错误信息]

### 汇总结果
[所有成功任务的整合结果]

### 统计
- 总耗时: Xm
- 平均每任务: Xs
- 成功率: Y%
- Token消耗: Z
```

## 注意事项
- 大规模任务注意Token消耗
- 监控失败率，过高时暂停检查
- 保存检查点，支持断点续传
