触发自我进化: 分析问题并更新系统配置。

## 进化触发: $ARGUMENTS

## 进化流程

### 1. 问题/经验分析
```
- 发生了什么?
- 根本原因是什么?
- 影响范围多大?
- 是否为重复问题?
```

### 2. 解决方案生成
```
- 如何防止再次发生?
- 需要更新哪些配置?
- 是否需要新增Agent或命令?
- 是否需要调整策略?
```

### 3. 配置更新

根据分析结果，可能更新以下文件:

**CLAUDE.md**:
- 添加新规则
- 调整决策权限
- 更新命令列表

**agents/*.md**:
- 优化Agent提示词
- 调整工具权限
- 更新能力描述

**commands/*.md**:
- 改进命令流程
- 添加新命令
- 修复命令问题

**memory/*.md**:
- 记录经验教训
- 更新错误模式
- 沉淀最佳实践

### 4. 验证更新
```
- 更新是否有效?
- 是否引入新问题?
- 是否需要回滚?
```

## 输出格式

```markdown
## 进化报告

### 触发原因
[问题/经验描述]

### 根因分析
[分析结果]

### 配置更新
| 文件 | 更新内容 | 理由 |
|------|----------|------|
| CLAUDE.md | ... | ... |
| memory/lessons-learned.md | 新增条目 | 记录经验 |

### 验证结果
[更新后的验证]

### 后续建议
[如果有进一步建议]
```

## 自动进化触发条件
- ❌ 任务失败或需要人工纠正
- ⚠️ 重复犯同类错误 (≥2次)
- 💡 发现更优的解决方案
- 🔄 完成复杂任务后的回顾
