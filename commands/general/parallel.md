并行执行多个独立任务。

## 任务列表
$ARGUMENTS

## 并行执行规则

### 1. 任务验证
- 检查任务是否真正独立 (无资源冲突)
- 验证任务格式正确
- 确认Agent可用

### 2. 并行配置
- 最大并行度: 5 (可根据任务复杂度调整)
- 超时设置: 根据任务复杂度自动设置
- 重试策略: 失败任务最多重试2次

### 3. 执行模式
```
同时启动多个Task工具调用:
- Task 1 → Agent A (后台)
- Task 2 → Agent B (后台)
- Task 3 → Agent C (后台)
...

等待所有任务完成或超时
```

### 4. 结果收集
- 使用AgentOutputTool收集各任务结果
- 处理失败的任务
- 整合成功的结果

## 输出格式

```markdown
## 并行执行报告

### 任务状态
| 任务 | Agent | 状态 | 耗时 |
|------|-------|------|------|
| Task1 | agent-a | ✅ 成功 | 5s |
| Task2 | agent-b | ✅ 成功 | 8s |
| Task3 | agent-c | ❌ 失败 | - |

### 成功结果
[各任务的输出]

### 失败分析
[失败任务的错误信息和建议]

### 总耗时
实际: Xs (串行预估: Ys, 节省: Z%)
```

## 注意事项
- 确保任务真正独立，避免资源冲突
- 监控Token消耗
- 处理部分失败的情况
