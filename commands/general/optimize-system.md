# 系统优化分析

调用 auto-optimizer Agent 分析系统性能，识别优化机会，生成优化方案。

## 用法

```bash
/optimize-system [target]
```

## 参数

- **target** (可选): 优化目标
  - 不指定 - 全面分析，识别所有优化机会
  - `cost` - 专注成本优化
  - `speed` - 专注速度优化
  - `quality` - 专注质量提升
  - `{agent_name}` - 优化特定Agent

## 示例

### 全面优化分析
```bash
/optimize-system
```

### 专注成本优化
```bash
/optimize-system cost
```

### 优化特定Agent
```bash
/optimize-system architect
```

## 输出

### 优化机会列表
包含:
- 优化项描述
- 优先级 (高/中/低)
- 预期收益
- 实施风险
- 具体方案

### 优化方案文档
保存至: `memory/optimization-proposals/YYYY-MM-DD.md`

包含:
- 详细分析
- 优化方案
- A/B测试计划
- 回滚策略

## 优化流程

### 1. 数据分析阶段
- 从 performance-monitor 获取性能数据
- 计算各项性能指标
- 识别性能异常和瓶颈

### 2. 机会识别阶段
- 模型选择优化机会
- Prompt优化机会
- 工作流优化机会
- 成本优化机会

### 3. 方案设计阶段
- 设计优化方案
- 制定A/B测试计划
- 评估风险和收益
- 准备回滚策略

### 4. 输出交付
- 生成优化建议清单
- 输出详细方案文档
- 提供实施指导

## 优化类型

### 模型选择优化
- 识别可降级到Haiku的场景
- 评估模型切换的影响
- 计算成本节省

### Prompt优化
- 长度优化 (移除冗余)
- 清晰度优化 (改进结构)
- 有效性优化 (增强效果)

### 工作流优化
- 并行化机会
- 缓存策略
- 流程简化

### 成本优化
- 识别高成本操作
- 推荐低成本替代
- 优化Token使用

## 实施建议

生成的优化方案通常包含:

### 立即可实施 (低风险)
- Prompt长度优化
- 引入缓存机制
- 并行化改造

### 需要测试验证 (中风险)
- 模型降级
- Prompt结构调整
- 工作流重组

### 需要审慎评估 (需讨论)
- 重大架构调整
- 功能简化
- 质量标准调整

## 验证优化效果

实施优化后，使用以下命令验证:

```bash
/agents auto-optimizer "验证优化方案 #{id} 的效果"
```

## 相关命令

- `/performance-report` - 查看性能报告
- `/agents auto-optimizer` - 直接调用优化器Agent

## 最佳实践

### 1. 定期执行
- 建议每周执行一次全面分析
- 月度执行深度优化评估

### 2. 优先级管理
- 优先实施高收益、低风险的优化
- 中等风险的需要A/B测试
- 高风险的需要充分讨论

### 3. 效果追踪
- 实施后持续监控指标
- 对比优化前后数据
- 及时调整或回滚

### 4. 知识积累
- 记录成功的优化模式
- 总结失败的教训
- 形成最佳实践

## 触发优化的时机

### 自动触发条件
- 连续3次超过性能阈值
- 周性能下降 > 20%
- 月成本超出预算

### 主动触发场景
- 引入新功能后
- 用户反馈质量问题
- 发现性能瓶颈
- 定期优化评估

## 注意事项

1. **数据要求**: 需要至少1周的性能数据才能生成有效建议
2. **实施风险**: 始终进行A/B测试验证效果
3. **回滚准备**: 确保所有优化都有回滚计划
4. **用户体验**: 优化不应影响用户满意度
