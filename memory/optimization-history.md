## 示例优化记录

### [2026-01-16] 优化方案 #001 - architect Prompt精简

#### 📋 背景
- **触发原因**: architect Token消耗持续偏高，超出阈值
- **当前状态**: 平均15,500 tokens/任务
- **目标阈值**: 12,000 tokens/任务
- **优先级**: 高
- **提出者**: auto-optimizer Agent
- **审批者**: 待定

#### 🔍 分析
- **根因**: prompt包含15个代码示例，总长度约8000 tokens
- **影响范围**: 所有architect任务
- **成本影响**: 每月额外成本 $45
- **数据支撑**:
  - 过去30天平均Token: 15,500
  - 阈值: 12,000
  - 超出比例: 29.2%

#### 💡 方案
1. **移除冗余示例**: 从15个减少到5个核心示例
   - 保留: 最具代表性的5个
   - 移除: 相似场景的冗余示例

2. **提取共享内容**: 将通用规范移至 `BEST-PRACTICES.md`
   - 代码规范
   - 命名约定
   - 架构模式

3. **精简指令**: 合并相似的指令步骤
   - 原: 12个详细步骤
   - 优化: 6个精简步骤

#### 🎯 预期效果
| 指标 | 当前 | 目标 | 改善幅度 |
|------|------|------|----------|
| 平均Token | 15,500 | 10,500-11,500 | -25% ~ -30% |
| 成功率 | 95.8% | ≥95% | 保持 |
| 用户评分 | 4.6 | ≥4.5 | 保持 |
| 月成本 | $155 | $120 | -$35 |

#### 🧪 A/B测试计划
- **测试组**: 10个新任务使用优化prompt
- **对照组**: 10个新任务使用原prompt
- **测试周期**: 3天 (2026-01-16 至 2026-01-19)
- **成功标准**:
  - Token降低 ≥20%
  - 成功率 ≥95%
  - 用户评分 ≥4.5
  - 无明显质量下降

#### 🔄 实施步骤
1. ✅ 创建优化后的prompt副本
2. ✅ 在测试环境验证语法和格式
3. ⏳ 启动A/B测试
4. ⏳ 每天收集数据并分析
5. ⏳ 根据结果决定推广或回滚

#### ⚠️ 风险评估
- **风险等级**: 低
- **主要风险**:
  1. 指令不够详细导致输出质量下降
  2. 缺少示例导致理解偏差
- **缓解措施**:
  1. 小范围测试 + 人工审核前10个输出
  2. 保留关键示例，确保覆盖核心场景
  3. 准备快速回滚机制

#### 🔙 回滚计划
- **触发条件**:
  - 成功率 <93%
  - 用户评分 <4.3
  - 质量问题超过3个
- **回滚方式**: 恢复原prompt配置
- **回滚时间**: <5分钟
- **回滚负责人**: 待指定

#### 状态: 📝 待实施

## 使用说明

### 添加新优化记录
1. 复制示例优化记录格式
2. 填写所有必需字段
3. 分配唯一ID (格式: #XXX)
4. 更新总览表格

### 更新优化状态
1. 定位到对应的优化记录
2. 更新状态标记
3. 添加测试结果或实际效果
4. 更新统计数据

### 提炼优化模式
1. 从成功的优化中识别共性
2. 提炼核心方法和步骤
3. 添加到优化模式库
4. 标注适用场景和预期收益

### 记录失败教训
1. 诚实记录失败的尝试
2. 深入分析失败原因
3. 提炼可避免的错误
4. 分享给团队
