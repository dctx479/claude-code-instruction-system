# 知识沉淀策略

## 核心理念

知识是Apollo系统自进化的燃料。通过持续沉淀、智能检索和动态应用知识，实现跨会话的能力积累和智能增长。

---

## 一、自动触发机制

### 1.1 触发条件矩阵

| 事件类型 | 触发时机 | 沉淀内容 | 优先级 |
|----------|----------|----------|--------|
| **任务完成** | 每次任务成功结束 | 解决方案、技术选型、实现模式 | 高 |
| **错误发生** | 捕获错误或异常 | 错误模式、根因分析、修复方案 | 最高 |
| **发现最佳实践** | 识别优秀模式 | 代码模式、设计原则、优化技巧 | 高 |
| **技术决策** | 做出关键选择 | 决策依据、权衡分析、结果评估 | 中 |
| **Agent使用** | Agent执行任务 | 性能数据、适用场景、配置策略 | 中 |
| **工作流执行** | 完成复杂流程 | 流程模板、协作模式、效率指标 | 高 |
| **配置更新** | 修改系统配置 | 变更原因、影响范围、回滚方案 | 中 |
| **学习新技术** | 掌握新知识 | 技术特性、使用方法、注意事项 | 中 |

### 1.2 自动执行流程

```
事件发生 → 知识提取 → 结构化处理 → 构建图谱 → 持久化
    ↓           ↓            ↓           ↓          ↓
 识别类型    提取关键点    实体关系化   添加连接   存入Neo4j
    ↓           ↓            ↓           ↓          ↓
 判断优先级  去重去噪     添加元数据   图谱融合   更新索引
```

### 1.3 智能沉淀判断

**必须沉淀 (自动执行)**:
- ✅ 解决了之前未见过的问题
- ✅ 发现系统bug或配置错误
- ✅ 完成复杂多步骤任务
- ✅ 使用新工具或技术
- ✅ 优化提升显著 (>20%)

**可选沉淀 (评估后决定)**:
- ⚪ 重复性简单任务
- ⚪ 常规代码修改
- ⚪ 已知问题的标准处理
- ⚪ 临时性实验和测试

**不应沉淀 (过滤掉)**:
- ❌ 一次性临时数据
- ❌ 敏感信息 (密码、密钥)
- ❌ 无价值的噪声信息
- ❌ 已充分记录的知识

```

属性:

  - architecture_type: 架构模式
  - tech_stack: 技术栈

关系:
  - (系统) -[包含]-> (组件)
  - (系统) -[使用]-> (技术)
```

## 三、记忆格式规范

### 3.1 实体定义标准

```json
{
  "entity": {
    "name": "实体唯一名称",
    "type": "实体类型",
    "properties": {
      "description": "描述",
      "category": "分类",
      "version": "版本",
      "created_at": "2026-01-16T10:00:00Z",
      "updated_at": "2026-01-16T10:00:00Z",
      "source": "来源",
      "confidence": 0.95,
      "custom_field": "自定义属性"
    },
    "metadata": {
      "project": "项目名",
      "author": "创建者",
      "tags": ["标签1", "标签2"]
    }
  }
}
```

**命名规范**:
- 使用描述性名称: `"Apollo自进化系统"` ✓
- 避免缩写: `"TypeScript"` ✓ 而非 `"TS"` ✗
- 统一语言: 中文实体用中文，英文实体用英文
- 唯一性: 同名实体使用限定词区分

### 3.2 关系定义标准

```json
{
  "relation": {
    "source": "源实体名称",
    "type": "关系类型",
    "target": "目标实体名称",
    "properties": {
      "strength": 0.8,
      "direction": "单向/双向",
      "created_at": "2026-01-16T10:00:00Z",
      "context": "关系上下文",
      "evidence": "支持证据"
    }
  }
}
```

**关系类型词汇表**:
- `使用` (uses)
- `包含` (contains)
- `依赖` (depends_on)
- `实现` (implements)
- `继承` (inherits)
- `关联` (related_to)
- `解决` (solves)
- `导致` (causes)
- `优于` (better_than)
- `替代` (replaces)

### 3.3 时序标记规范

**时间戳格式**:
- ISO 8601标准: `2026-01-16T10:00:00Z`
- 包含时区信息
- 精确到秒级

**时序元数据**:
```json
{
  "temporal": {
    "created_at": "创建时间",
    "updated_at": "更新时间",
    "valid_from": "生效起始时间",
    "valid_until": "生效结束时间",
    "version_history": [
      {"version": "1.0", "date": "2026-01-01"},
      {"version": "2.0", "date": "2026-01-16"}
    ]
  }
}
```

### 3.4 事件记录标准

```json
{
  "episode": {
    "content": "事件详细描述",
    "timestamp": "2026-01-16T10:00:00Z",
    "type": "任务完成/错误发生/决策制定",
    "entities": ["相关实体1", "相关实体2"],
    "outcome": "成功/失败/部分成功",
    "lessons_learned": "经验教训",
    "artifacts": ["生成物1", "生成物2"]
  }
}
```

## 四、检索策略

### 4.1 检索模式

**语义搜索** (默认):
- 用途: 模糊匹配、概念检索
- 优势: 理解意图、容错性强
- 示例: "如何优化性能" → 找到性能相关的所有知识

**精确查询**:
- 用途: 查找特定实体
- 优势: 快速准确
- 示例: 查询实体 "Apollo系统"

**图遍历**:
- 用途: 探索关联知识
- 优势: 发现隐藏联系
- 示例: 从 "Apollo系统" 出发，遍历所有使用的技术

**时序过滤**:
- 用途: 历史分析、版本追踪
- 优势: 理解演化
- 示例: 查询2026年1月的所有决策

### 4.2 检索优化技巧

**查询构造**:
```
❌ 差: "Agent"
✓ 好: "Agent编排策略在复杂任务中的应用"

❌ 差: "错误"
✓ 好: "TypeScript类型错误的常见模式和解决方案"

❌ 差: "性能"
✓ 好: "React应用性能优化的最佳实践"
```

**组合检索**:
```markdown
1. 语义搜索找到候选集
2. 实体类型过滤缩小范围
3. 时间范围限定版本
4. 关系遍历扩展结果
```

**结果排序**:
- 相关性得分 (默认)
- 时间新旧 (最新/最早)
- 使用频率 (热门)
- 置信度 (可靠性)

### 4.3 智能推荐

**上下文感知**:
- 根据当前任务推荐相关知识
- 基于历史行为预测需求
- 识别知识缺口主动提醒

**关联发现**:
- 自动关联相似问题
- 推荐替代解决方案
- 发现知识间的隐藏联系

## 六、隐私与安全

### 6.1 敏感信息处理

**严禁存储**:
- ❌ 密码和API密钥
- ❌ 个人身份信息 (PII)
- ❌ 财务敏感数据
- ❌ 商业机密
- ❌ 用户隐私数据

**脱敏后存储**:
- 🔒 错误日志 (移除路径、用户名)
- 🔒 配置信息 (替换真实值为占位符)
- 🔒 示例数据 (使用假数据)

**可安全存储**:
- ✅ 公开文档和教程
- ✅ 技术方案和设计
- ✅ 代码模式和最佳实践
- ✅ 工作流程和流程图

### 6.2 访问控制

**本地优先**:
- Neo4j部署在本地
- 数据不离开本地环境
- 完全用户控制

**认证机制**:
- 数据库密码保护
- 访问日志记录
- 异常访问告警

### 6.3 数据备份

**备份策略**:
```bash
# 每日自动备份
neo4j-admin dump --database=graphiti --to=/backup/graphiti-$(date +%Y%m%d).dump

# 每周完整备份
tar -czf graphiti-backup-$(date +%Y%m%d).tar.gz /neo4j/data

# 关键变更即时备份
neo4j-admin dump --database=graphiti --to=/backup/graphiti-before-change.dump
```

**恢复测试**:
- 每月测试备份恢复
- 验证数据完整性
- 记录恢复时间

## 七、集成Apollo系统

### 7.1 自进化协议集成

```
错误发生 → 自动提取错误模式 → 存储到Graphiti
         ↓
      根因分析 → 创建因果关系 → 连接到知识图谱
         ↓
      解决方案 → 建立解决关系 → 丰富知识网络
```

**实施方式**:
- Hook到错误处理流程
- 自动触发知识提取
- 异步存储不阻塞主流程

### 7.2 Agent驾驭集成

```
Agent执行 → 记录性能数据 → 存储为实体属性
         ↓
      策略选择 → 记录决策依据 → 建立策略关系
         ↓
      任务结果 → 评估有效性 → 更新策略评分
```

**数据收集**:
- Agent执行时间
- 资源消耗
- 成功率
- 错误率
- 适用场景

### 7.3 记忆系统同步

```
memory/*.md ←→ Graphiti
      ↓              ↓
  结构化文本      知识图谱
      ↓              ↓
  易于阅读       易于检索
      ↓              ↓
   定期同步 ←────→ 双向增强
```

**同步策略**:
- 每日同步: memory/*.md → Graphiti (结构化导入)
- 按需导出: Graphiti → memory/*.md (生成报告)
- 冲突解决: 以Graphiti为准，手动确认
