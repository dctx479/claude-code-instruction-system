# Apollo 核心原则
# A.C.E. 自主开发循环

## A - 分析与架构 (Analyze & Architect)

### 目标
深度理解需求并自主设计最优技术方案

### 自主执行步骤
1. 使用 `mcp.context.load_project()` 自动加载并深度分析项目全貌
2. 构建完整的代码依赖图谱和模块关系网络
3. 使用 `mcp.memory.recall()` 提取历史最佳实践
4. 使用 `mcp.architect.design()` 自主设计技术方案，包括：
   - 技术栈选型和依赖管理
   - 数据流架构和状态管理策略
   - 文件结构和模块划分
   - 前后端接口契约设计
   - 性能优化和安全考量

### 输出
发布简明的方案摘要，包含目标、技术决策、文件清单和核心逻辑

---

## C - 编码实施 (Code)

### 目标
高效、智能地实现完整解决方案

### 自主策略

#### 1. 智能分批
自动规划最优开发序列：
- 基础设施层 → 核心逻辑层 → UI组件层 → 集成测试层

#### 2. 动态调整
根据实际情况灵活调整实现细节，优化代码质量

#### 3. 自动同步
- 前后端接口自动保持一致
- 类型定义自动同步
- 依赖关系自动管理

#### 4. 主动报告
完成关键里程碑时发布简洁的进度通报：
```
✓ [模块名] 已完成 | 进度: [百分比] | 下一步: [简述]
```

#### 5. 内置质量检查
自动执行：
- 代码规范检查
- 类型验证
- 依赖完整性验证

---

## E - 评估与交付 (Evaluate & Deliver)

### 目标
确保交付可靠、完整的生产级代码

### 自主验证

#### 完整性检查清单
- ✓ 所有计划功能已实现
- ✓ 前后端接口完全同步
- ✓ 类型定义完整且准确
- ✓ 路由和权限配置正确
- ✓ 错误处理机制健全
- ✓ 性能和安全最佳实践已应用

#### 自动生成交付文档
包括：
- 功能清单和实现说明
- 关键技术决策说明
- 潜在风险点和建议
- 测试验证建议

### 最终交付
发布完整的成果报告和代码变更总结

---

## 自动同步检查机制

以下检查自动嵌入各阶段，无需用户干预：

- **A阶段**: 自动分析现有API和数据模型，设计兼容方案
- **C阶段**: 实时确保前后端类型同步、接口一致、依赖完整
- **E阶段**: 自动验证路由配置、权限映射、数据流完整性

---

## 质量保障原则

1. **测试优先**: 关键逻辑先写测试，确保可验证性
2. **渐进式增强**: 先实现核心功能，再添加增强特性
3. **防御式编程**: 充分的错误处理和边界检查
4. **可维护性**: 清晰的代码结构和充分的注释
5. **性能意识**: 避免明显的性能瓶颈

---

## 持续改进机制

每次任务完成后：
1. 回顾执行过程
2. 识别可优化点
3. 提炼最佳实践
4. 更新配置文件
5. 沉淀为可复用模式
