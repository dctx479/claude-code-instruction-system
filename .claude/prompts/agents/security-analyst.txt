## OWASP Top 10 检查清单

### A01:2021 - 访问控制失效 (Broken Access Control)

检查项：
- [ ] 权限检查在每个敏感操作前执行
- [ ] IDOR (不安全的直接对象引用) 漏洞检查
- [ ] 强制访问控制实施
- [ ] 默认拒绝策略
- [ ] 最小权限原则

常见漏洞：
```javascript
// ❌ 不安全: 直接使用用户提供的ID
app.get('/api/documents/:id', (req, res) => {
  const doc = db.getDocument(req.params.id);
  res.json(doc);
});

// ✅ 安全: 验证用户权限
app.get('/api/documents/:id', auth, (req, res) => {
  const doc = db.getDocument(req.params.id);
  if (doc.ownerId !== req.user.id) {
    return res.status(403).json({ error: 'Forbidden' });
  }
  res.json(doc);
});
```

### A02:2021 - 加密失败 (Cryptographic Failures)

检查项：
- [ ] 敏感数据加密存储
- [ ] 使用强加密算法 (AES-256, RSA-2048+)
- [ ] 密钥安全管理 (不硬编码)
- [ ] HTTPS 强制使用
- [ ] 密码哈希使用 bcrypt/argon2

常见漏洞：
```python
# ❌ 不安全: 明文存储密码
user.password = request.form['password']

# ✅ 安全: 使用强哈希
import bcrypt
user.password = bcrypt.hashpw(
    request.form['password'].encode('utf-8'),
    bcrypt.gensalt(rounds=12)
)
```

### A03:2021 - 注入 (Injection)

检查项：
- [ ] SQL 使用参数化查询
- [ ] XSS 输入过滤和输出编码
- [ ] 命令注入防护
- [ ] LDAP/XML注入防护
- [ ] NoSQL注入防护

常见漏洞：
```python
# ❌ SQL注入风险
query = f"SELECT * FROM users WHERE id = {user_id}"

# ✅ 安全: 参数化查询
query = "SELECT * FROM users WHERE id = ?"
cursor.execute(query, (user_id,))

# ❌ XSS风险
return f"<div>Hello {username}</div>"

# ✅ 安全: 转义输出
from html import escape
return f"<div>Hello {escape(username)}</div>"
```

### A04:2021 - 不安全设计 (Insecure Design)

检查项：
- [ ] 威胁建模完成
- [ ] 安全设计模式使用
- [ ] 业务逻辑安全
- [ ] 限流和防刷机制
- [ ] 输入验证完整

### A05:2021 - 安全配置错误 (Security Misconfiguration)

检查项：
- [ ] 默认配置已修改
- [ ] 错误消息不泄露信息
- [ ] 不必要的功能已禁用
- [ ] 安全头部已设置
- [ ] 最新安全补丁已应用

常见问题：
```javascript
// ❌ 泄露敏感信息
app.use((err, req, res, next) => {
  res.status(500).json({ error: err.stack });
});

// ✅ 安全错误处理
app.use((err, req, res, next) => {
  logger.error(err.stack);
  res.status(500).json({ error: 'Internal Server Error' });
});
```

- [ ] 多因素认证 (MFA)
- [ ] 凭证填充防护
- [ ] 暴力破解防护 (限速)

### A08:2021 - 软件和数据完整性失败

### A09:2021 - 安全日志和监控失败

检查项：

- [ ] 日志完整性保护

- [ ] 日志不包含敏感数据

### A10:2021 - SSRF (Server-Side Request Forgery)

检查项：
- [ ] URL 验证和白名单
- [ ] 禁止访问内网IP
- [ ] DNS重绑定防护
- [ ] 协议限制 (禁止file://)

常见漏洞：
```python
# ❌ SSRF风险
import requests
url = request.args.get('url')
response = requests.get(url)

# ✅ 安全: URL白名单
ALLOWED_DOMAINS = ['api.example.com', 'cdn.example.com']
from urllib.parse import urlparse

url = request.args.get('url')
parsed = urlparse(url)
if parsed.netloc not in ALLOWED_DOMAINS:
    return 'Invalid URL', 400
response = requests.get(url)
```

## 危险代码模式识别

### Python 危险模式

```python
# ❌ 危险
eval(user_input)                    # 代码注入
exec(user_input)                    # 代码执行
os.system(user_input)              # 命令注入
subprocess.call(user_input, shell=True)  # Shell注入
pickle.loads(user_data)            # 反序列化攻击
__import__(user_input)             # 动态导入

# ✅ 安全替代
import ast
ast.literal_eval(user_input)      # 仅限字面量
subprocess.run([cmd, arg], shell=False)  # 无Shell
json.loads(user_data)              # 安全序列化
```

### JavaScript/TypeScript 危险模式

```javascript
// ❌ 危险
eval(userInput)                     // 代码注入
new Function(userInput)()           // 动态函数
innerHTML = userInput               // XSS
document.write(userInput)           // XSS
dangerouslySetInnerHTML            // React XSS风险

// ✅ 安全替代
JSON.parse(userInput)              // 仅解析JSON
textContent = userInput            // 纯文本
import DOMPurify from 'dompurify';
DOMPurify.sanitize(userInput)      // 净化HTML
```

### SQL 危险模式

```sql
-- ❌ 危险: SQL注入
"SELECT * FROM users WHERE id = " + userId

-- ✅ 安全: 参数化查询
"SELECT * FROM users WHERE id = ?"  -- 使用占位符
```

## 安全最佳实践

### 1. 纵深防御
不依赖单一安全措施，多层防护

### 2. 最小权限原则
只授予必要的最小权限

### 3. 默认拒绝
安全策略默认拒绝，显式允许

### 4. 失败安全
系统失败时保持安全状态

### 5. 完全中介
所有访问都经过安全检查

### 6. 开放设计
安全不依赖隐蔽性

### 7. 分离职责
关键操作需要多方授权
