# Question Refiner Skill

## 契约定义

### What（输入/输出）

**输入**: 用户原始研究查询（自然语言）

**输出**: Structured Research Prompt，包含：
- 核心问题（用户真正想知道什么）
- 研究范围（时间/地域/行业边界）
- 信息类型（数据/趋势/对比/案例）
- 前提假设（已知条件和约束）
- 排除条件（不关心的方面）

### When Done（验收标准）

- 输出为标准化 Structured Research Prompt 格式
- 核心问题明确且可操作
- 范围界定清晰，无歧义
- 输出 ≤300 字
- 未改变用户原始意图

### What NOT（边界约束）

- **禁止执行搜索**：仅做查询精炼，不执行任何 web_search/web_fetch
- **禁止改变用户意图**：精炼是澄清而非重定向
- **禁止过度追问**：最多 3 个澄清问题
- **禁止生成报告**：输出仅为 Structured Research Prompt

---

## 何时使用此 Skill

当研究类 Skill 接收到模糊或不完整的查询时，作为**可选前置网关**使用。

**触发条件**（满足任一即触发）：
- 查询长度 <10 字且无法判断研究类型
- 查询范围过于宽泛（如"研究一下 AI"）
- 用户显式请求帮助澄清研究方向

**不触发条件**（满足任一即跳过）：
- 查询已包含明确主题、范围和期望输出
- 用户已指定研究深度（快速/标准/深度）
- 查询为直接事实型问题（straightforward）

## 7 项验证清单

对用户查询逐项检查：

| # | 检查项 | 说明 | 示例（通过） | 示例（未通过） |
|---|--------|------|-------------|---------------|
| 1 | **意图明确** | 能判断用户想知道什么 | "对比 React 和 Vue 的性能" | "看看前端" |
| 2 | **范围界定** | 有时间/地域/行业等边界 | "2025 年中国市场" | "全球所有情况" |
| 3 | **信息类型** | 能判断需要数据/趋势/案例 | "需要市场份额数据" | "了解一下" |
| 4 | **可操作性** | 可转化为具体搜索任务 | "GPT-4o 的定价策略" | "AI 怎么样" |
| 5 | **无歧义** | 关键术语无多义性 | "Apple 公司股价" | "Apple 的表现" |
| 6 | **粒度适当** | 不过宽也不过窄 | "新能源汽车市场趋势" | "人类科技发展" |
| 7 | **前提声明** | 已知条件已说明 | "假设预算 100 万" | （无前提也可通过） |

---

## 快速通过机制

**规则**：7 项中 ≥5 项通过 → **跳过追问，直接精炼输出**

这避免了对已清晰查询增加不必要的交互摩擦。流程：

```
用户查询 → 7 项检查
  ├─ ≥5 通过 → 直接生成 Structured Research Prompt（零追问）
  └─ <5 通过 → 引导式澄清（仅针对未通过项，最多 3 个问题）
```

---

## 引导式澄清

仅当通过项 <5 时启用。规则：

1. **仅针对未通过项**提问，不重复已明确的信息
2. **最多 3 个问题**，优先级：意图 > 范围 > 信息类型
3. **每个问题提供 2-3 个选项**，降低用户回答成本
4. 用户回答后立即生成 Structured Research Prompt

**澄清问题模板**：

```
关于您的研究 "[原始查询]"，我需要确认几点：

1. [未通过项 1]：您更关注 A、B 还是 C？
2. [未通过项 2]：时间范围是 X 还是 Y？
3. [未通过项 3]：需要 数据对比 还是 趋势分析？
```

---

## Structured Research Prompt 输出模板

```markdown
## Structured Research Prompt

**核心问题**: [用户真正想知道什么，一句话]

**研究范围**:
- 时间: [具体时间范围，如 "2024-2025"]
- 地域: [地域范围，如 "全球" / "中国市场"]
- 行业: [行业范围，如 "新能源汽车"]

**信息类型**: [数据统计 / 趋势分析 / 方案对比 / 案例研究 / 技术评估]

**前提假设**:
- [已知条件 1]
- [已知条件 2]（如无则标注"无特定前提"）

**排除条件**:
- [不关心的方面]（如无则标注"无特定排除"）

**期望产出**: [报告 / 简报 / 数据表 / 对比矩阵]
```

此格式可被所有 research 类 Skill 直接消费，替代原始自然语言查询。

## 集成协议

| 下游 Skill | 集成方式 | 说明 |
|------------|---------|------|
| **deep-research** | Phase 0 可选前置 | 模糊查询时触发，输出替代原始查询进入 Research Brief |
| **exa-research** | 查询预处理 | 精炼后的核心问题用作 Exa 语义搜索输入 |
| **brightdata-research** | 查询预处理 | 精炼后的范围约束用于限定电商平台和品类 |
| **amazon-analyse** | 不集成 | 该 Skill 输入为 ASIN，非自然语言查询 |
| **social-media-research** | 查询预处理 | 精炼后的信息类型用于选择数据采集维度 |

## 使用示例

### 示例 1: 模糊查询（需要澄清）

**用户**: "研究一下 AI"

**验证结果**: 1/7 通过（仅"前提声明"通过，因无前提也可）

**澄清问题**:
1. 您更关注 AI 的哪个方面：技术进展、市场规模 还是 应用场景？
2. 时间范围：最新动态（2025-2026）还是 历史演变？
3. 需要：数据报告 还是 趋势分析？

**用户回答**: "AI 编程助手的市场规模，最新的"

**输出**:
```
核心问题: AI 编程助手市场在 2025-2026 年的市场规模和增长趋势
研究范围: 时间 2025-2026 / 地域 全球 / 行业 AI 开发工具
信息类型: 数据统计 + 趋势分析
前提假设: 无特定前提
排除条件: 无特定排除
期望产出: 结构化报告
```

### 示例 2: 宽泛查询（部分澄清）

**用户**: "对比一下主流云服务商"

**验证结果**: 4/7 通过（意图、可操作性、无歧义、粒度通过）

**澄清问题**:
1. 关注哪些维度：定价、性能、生态 还是 全面对比？

**用户回答**: "主要看定价和性能"

**输出**:
```
核心问题: 主流云服务商（AWS/Azure/GCP）的定价和性能对比
研究范围: 时间 2025 最新 / 地域 全球 / 行业 云计算
信息类型: 方案对比
前提假设: 无特定前提
排除条件: 生态和市场份额
期望产出: 对比矩阵 + 报告
```

### 示例 3: 已清晰查询（快速通过）

**用户**: "帮我调研 2025 年中国新能源汽车市场的销量数据和主要厂商市场份额"

**验证结果**: 6/7 通过 → **快速通过，零追问**

**直接输出**:
```
核心问题: 2025 年中国新能源汽车市场的销量数据和厂商市场份额分布
研究范围: 时间 2025 / 地域 中国 / 行业 新能源汽车
信息类型: 数据统计
前提假设: 无特定前提
排除条件: 无特定排除
期望产出: 数据报告（含数据表）
```
