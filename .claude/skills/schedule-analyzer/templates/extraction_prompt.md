# 课表图片标准化提取 Prompt

## 使用方法

将以下内容作为 Prompt，配合课表截图使用。每次可批量处理 7-10 人。

---

## Prompt 模板

```
请仔细查看这张课表图片，提取所有课程信息。

## 输出格式

每门课一行，使用竖线分隔：
姓名|星期|大节|周次范围

## 命名规范

- **星期**: 周一、周二、周三、周四、周五、周六、周日
- **大节**: 第一大节、第二大节、第三大节、第四大节、第五大节

## 小节 → 大节映射

如果课表显示的是小节（第1节、第2节...），请按以下规则转换为大节：
- 第1-2节 → 第一大节
- 第3-4节 → 第二大节
- 第5-6节 → 第三大节
- 第7-8节 → 第四大节
- 第9-10节 / 第9-11节 → 第五大节

## 周次格式

保留原始格式，不要展开：
- 范围: 1-16
- 单周: 1-15单
- 双周: 2-16双
- 逗号分隔: 1-8,10-17
- 混合: 1-5,7-15单

## 拆分规则

- 同一大节如有多门课（不同周次），拆分为多行
  例如: 周二第三大节有A课(1-8周)和B课(9-16周) → 两行

## 提取要求

1. 逐列扫描: 从周一到周日，逐列逐行提取
2. 不遗漏: 检查每个时段格子，包括底部可能的附加课程表
3. 周次要完整: 确保周次范围准确，不要猜测
4. 图片模糊时: 在周次后标注 [UNCLEAR]

## 示例输出

张三|周一|第一大节|1-16
张三|周一|第二大节|1-15单
张三|周一|第三大节|5-14
张三|周二|第一大节|1-16
张三|周二|第三大节|1-8
张三|周二|第三大节|10-16双
```

---

## 批量提取注意事项

1. **每批 7-10 人**：避免单次提取过多导致遗漏
2. **同班优先分批**：同班学生放在一起便于交叉校验
3. **提取后校验**：
   - 每人应有 10-35 条记录（少于 10 或超过 35 需复查）
   - 同班学生应共享部分课程条目
   - 检查是否有遗漏的星期或大节

## 常见问题

| 问题 | 处理方式 |
|------|----------|
| 课表显示小节而非大节 | 按上方映射规则转换 |
| 同一格有两门课 | 拆分为两行，各自标注周次 |
| 底部有单独的课程表 | 一并提取，不要遗漏 |
| 图片模糊看不清周次 | 标注 `[UNCLEAR]`，后续人工确认 |
| 课程跨越两个大节 | 拆分为两行，例如第5-8节 → 第三大节 + 第四大节 |
| 周六/周日有课 | 正常提取，不要跳过 |
