# 课表空课查找统计 Skill

## 契约定义

### What（输入/输出）

**输入**：
| 参数 | 类型 | 必需 | 说明 |
|------|------|------|------|
| 课表图片目录 | 目录路径 | 是 | 包含学生课表截图的目录 |
| 分组定义 | JSON / Excel / 手动指定 | 是 | 成员分组信息 |
| 教学周数 | 整数 | 否 | 默认 17 周 |
| config.json | 文件路径 | 否 | 完整配置（含分组、阈值等） |

**输出**：
| 文件 | 说明 |
|------|------|
| `all_weeks.txt` | 中间数据，pipe 格式：`姓名\|星期\|大节\|周次范围` |
| `课表汇总.md` | 最终 Markdown 报告，含个人表 + 分组统计 + 周次分析 + 建议 |

### When Done（验收标准）

- 所有成员课表图片均已提取，无遗漏（缺失成员有告警）
- `all_weeks.txt` 格式正确，每人条目数在合理范围（10-35 条）
- `课表汇总.md` 包含：个人课表摘要 + 分组统计表 + 黄金时段表 + 周次差异分析 + 建议
- 黄金时段识别结果有具体时段和无课人数比例（≥88% 阈值）
- 周次差异分析标注了全学期视图与逐周视图的差异
- 报告中的建议至少包含 1 个可直接使用的活动安排时段

### What NOT（边界约束）

- **禁止单人分析**：单人课表直接看图即可，不需要此 Skill
- **禁止非大学课表格式**：中小学、培训机构排期格式不兼容
- **禁止跳过质量检查**：条目数异常（<10 或 >35）必须告警，不能静默通过
- **禁止编造提取结果**：模糊图片标注 `[UNCLEAR]`，不猜测内容
- **禁止忽略周次范围**：单双周/逗号分隔等格式必须正确解析，不能当全周处理
- **禁止跳过交叉校验**：同班同学应共享部分课程，差异过大时需告警

## 核心能力

### 1. Vision 批量提取
- 大节 / 小节双格式自动识别
- 逐列逐行提取，保证完整性
- 同一大节多门课拆分
- 底部附加课程表检查
- 模糊图片标注 `[UNCLEAR]`

### 2. 周次范围解析
- 标准范围：`1-16`
- 单周：`1-15单` → {1,3,5,...,15}
- 双周：`2-16双` → {2,4,6,...,16}
- 逗号分隔：`1-8,10-17`
- 混合：`1-5,7-15单`

### 3. 分组无课统计
- 全学期视图（任意周有课即标记有课）
- 逐周精确统计（考虑周次范围差异）
- 全学期 vs 逐周差异分析
- 分组独立统计 + 全体汇总

### 4. 黄金时段发现
- 可配置无课人数阈值（默认 88%）
- 排除周末
- 按组 / 全体分别发现
- 重点关注特定大节（可配置）

## 四阶段工作流

### 阶段一：准备 (Prepare)

1. 扫描图片目录，列出所有课表图片
2. 解析或创建分组定义
3. 生成 `config.json`：

```json
{
  "groups": {
    "人工智能": ["白梦娜", "杜怡萌", "..."],
    "网络安全": ["陈鸿名", "程万洋", "..."]
  },
  "total_weeks": 17,
  "focus_periods": ["第五大节"],
  "golden_threshold": 0.88,
  "title": "XX社团 XXXX学年第X学期",
  "exclude_weekends": true,
  "significant_change_threshold": 10
}
```

4. 确认成员名单完整性

### 阶段二：提取 (Extract)

1. 加载标准化提取 Prompt（`templates/extraction_prompt.md`）
2. 并行批量读取图片（每批 7-10 人），使用 Vision 能力
3. 输出 pipe 格式数据到 `all_weeks.txt`
4. 质量检查：
   - 每人条目数异常检测（<10 或 >35 告警）
   - 同班交叉校验（同班同课应有相同条目）
   - 星期 / 大节命名规范校验

### 阶段三：分析 (Analyze)

运行 Python 分析脚本：

```bash
python scripts/schedule_analyzer.py <all_weeks.txt> <config.json> <output.md>
```

脚本自动完成：
- 加载数据，模糊匹配姓名
- 完整性校验（缺失成员告警）
- 全学期统计
- 逐周统计 + 差异分析
- 黄金时段发现
- 全时段显著变化检测

### 阶段四：报告 (Report)

生成 Markdown 报告，包含以下章节：

1. **个人课表摘要**（每人有课时段数）
2. **分组统计表**（全学期 + 各周第五大节无课人数矩阵）
3. **黄金时段表**（高无课率时段，含有课者名单）
4. **周次差异分析**（与全学期视图的差异详情）
5. **全时段显著变化概要**（无课人数变化 ≥ 阈值的时段）
6. **建议**（推荐的活动安排时段）

## 准确度策略

### 提取阶段
1. **逐列逐行提取 Prompt**：要求 Vision 模型按周一到周日逐列扫描
2. **大节/小节自动检测映射**：
   - 第1-2节 → 第一大节
   - 第3-4节 → 第二大节
   - 第5-6节 → 第三大节
   - 第7-8节 → 第四大节
   - 第9-10节/第9-11节 → 第五大节
3. **同班交叉校验**：同一班级的学生应共享部分课程
4. **条目数量异常检测**：少于 10 条或超过 35 条时告警
5. **底部附加课程表检查**：有些课表图片底部有额外课程

### 分析阶段
1. **模糊姓名匹配**：子串匹配处理提取时的微小差异
2. **周次范围交叉验证**：检测不合理的周次范围（如超过总周数）
3. **统计一致性**：全学期无课人数 ≤ 任意周无课人数
