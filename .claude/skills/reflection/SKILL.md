# 任务复盘提炼器 (Reflection)

> 把刚完成的任务变成可复用的知识资产：萃取经验 → 两步链写入 → 评估 Skill 价值 → 链式触发 /neat。

## 设计动机

任务完成后，经验往往以两种方式消失：

| 消失方式 | 后果 |
|---------|------|
| 直接关窗口，不复盘 | 下次遇到同类问题从零开始 |
| 随手写几条"注意事项" | 泛化无案例，写了等于没写 |

本 Skill 对治这两种反模式：强制要求 before/after 示例 + 案例引用，并通过两步链 Ingest 防止知识库膨胀。

---

## What（输入/输出）

### 输入

| 输入项 | 必需 | 描述 |
|--------|------|------|
| **本次对话上下文** | 是 | 隐式输入，无需额外提供 |
| **任务描述** | 否 | 用户可补充"这次主要做了 X" |
| **关注点提示** | 否 | 用户可指定"重点看踩坑部分" |

### 输出

| 输出项 | 格式 | 描述 |
|--------|------|------|
| **候选经验条目** | Markdown 列表 | 3-5 条，含 before/after + 案例引用 |
| **Skill 提取评估** | 一段结论 | 是否满足提取阈值，推荐下一步 |
| **写入结果** | 文件变更 | 用户确认后写入 lessons-learned.md |
| **/neat 触发提示** | 一行提示 | 提醒用户收尾时调用 /neat |

---

## How（四步执行流程）

### Step 1 — 萃取候选经验

回顾本次对话，识别以下类型的事件：

| 事件类型 | 识别信号 | 示例 |
|---------|---------|------|
| **踩坑** | 尝试失败 → 修正 → 成功 | 第一次用 A 方法失败，改用 B 方法成功 |
| **发现** | 意外有效的做法 | 某个参数组合效果出乎意料好 |
| **决策** | 在多个方案中做了选择 | 选 Light 而非 Heavy，原因是… |
| **约束** | 发现了隐性限制 | Windows 路径不能用 `./`，必须用绝对路径 |

**萃取标准**（缺一不可）：
1. **可复用** — 下次遇到类似场景能直接用
2. **有案例** — 能关联到本次对话的具体操作/文件/错误
3. **有对比** — 能写出 before（错误做法）或 without（没有这条经验时的状态）

**禁止写入**：
- "要注意配置文件格式"（无具体场景）
- "可能会遇到路径问题"（无案例引用）
- 一次性特殊情况（无复用价值）

### Step 2 — 两步链 Ingest（写入前必做）

**不要直接写文件**，先执行分析：

**Step 2a — 分析（Analyze）**

```
对每条候选经验，执行：
1. Grep memory/lessons-learned.md 搜索关键词
2. 找出语义相关的已有条目（1-3 条）
3. 判断：
   - 重叠度高（>70%）且无新信息 → 标记"仅追加案例引用"
   - 重叠度中等 → 标记"合并到现有条目"
   - 与已有条目矛盾 → 标记"写新条目 + 旧条目加废弃标记"
   - 全新事实 → 标记"新建条目"
```

**Step 2b — 生成（Generate）**

展示分析结果给用户确认，格式：

```markdown
### 候选经验 #1
**内容**: [一句话描述]
**Before**: [错误做法或缺失状态]
**After**: [正确做法]
**案例引用**: [本次对话中的具体操作/文件/错误]
**写入策略**: [新建 / 合并到 #L042 / 仅追加案例到 #L015 / 废弃 #L023 并新建]
**置信度**: [UNVERIFIED]（新记录默认）
```

用户确认后，按策略写入 `memory/lessons-learned.md`。

### Step 3 — Skill 提取评估

对本次任务整体评估是否值得提取为 Skill：

| 评估维度 | 阈值 | 本次情况 |
|---------|------|---------|
| **重复模式** | 同类流程 ≥3 次 | [填写] |
| **流程复杂度** | ≥5 个有序步骤 | [填写] |
| **领域知识** | 含隐性经验/方法论 | [填写] |
| **复用价值** | 未来 1 个月内可能再用 | [填写] |

**结论**：
- 满足 ≥2 项 → 建议调用 `/skill-creator` 提取 Skill
- 满足 1 项 → 可选，视用户意愿
- 满足 0 项 → 不需要，经验写入 lessons-learned 即可

### Step 4 — 链式收尾

复盘完成后，输出一行提示：

```
✅ 复盘完成。建议接着调用 /neat 同步 docs/CLAUDE.md/memory 三层知识，防止跨会话失同步。
```

如用户同意，直接触发 `/neat`。

## When Done（验收标准）

### 必须满足

1. **有案例引用** — 每条写入的经验都关联了本次对话的具体操作/文件/错误
2. **两步链已执行** — 写入前已做 Grep 去重分析，无重复条目新建
3. **Skill 评估已输出** — 明确说明是否建议提取 Skill 及理由
4. **用户已确认** — 展示候选条目后等待用户确认，不自动写入

### 建议满足

- 写入条目数 ≤5（宁少勿滥，质量优先）
- 每条经验有明确的 before/after 对比
- 已提示用户调用 /neat

---

## What NOT（边界约束）

🚫 **不做的事**：

1. **不自动写入** — 必须展示候选条目，等用户确认后才写文件
2. **不写泛化经验** — 无案例引用的条目一律拒绝写入
3. **不替代 /neat** — 本 Skill 只处理 lessons-learned，三层同步由 /neat 负责
4. **不强制提取 Skill** — Skill 提取是建议，不是强制步骤
5. **不修改代码** — 只操作 memory/ 目录下的知识文件

🚫 **不该触发的场景**：

- 单轮简单问答（无新事实产生）
- 任务仍在进行中（应在收尾时调用）
- 用户明确说"不需要复盘"

---

## 调用时机

| 场景 | 是否调用 | 说明 |
|------|---------|------|
| 复杂任务完成后 | ✅ 强烈建议 | 主用法，萃取可复用经验 |
| 解决重大 bug 后 | ✅ 建议 | 踩坑经验最有复用价值 |
| autopilot/ralph 结束 | ✅ 建议 | 配合自进化协议 |
| 发现更优方案后 | ✅ 建议 | 记录决策依据 |
| 单轮问答 | ❌ 不需要 | 无新事实产生 |
| 任务中段 | ❌ 不需要 | 等任务完整收尾再做 |

## 与现有体系的关系

```
任务完成
    │
    ↓
[/reflection]          ← 萃取经验，写入 lessons-learned（本 Skill）
    │
    ├── 满足 Skill 提取阈值？
    │       ↓ 是
    │   [/skill-creator]   ← 把经验封装成可复用 Skill
    │
    ↓
[/neat]                ← 同步 docs/CLAUDE.md/memory 三层知识
    │
    ↓
下一会话从干净状态开始
```

**协同 Skill / 文档**：
- `.claude/skills/neat/SKILL.md` — 下游，负责三层知识同步
- `.claude/skills/skill-creator/SKILL.md` — 可选下游，把经验封装成 Skill
- `docs/KNOWLEDGE-COMPOUNDING-GUIDE.md` — 两步链 Ingest 协议的详细说明
- `memory/lessons-learned.md` — 写入目标文件
- `CLAUDE.md` 第一节 — 自进化协议（本 Skill 是其"半自动复盘流程"的执行载体）

## 实例

### 触发：用户说"/reflection"（刚完成 Windows hooks 路径修复）

```
Step 1 萃取候选经验:
  事件类型: 踩坑（第一次用 ./script.sh 失败，改用绝对路径成功）
  候选条目:
    - Windows hooks 不支持 ./script.sh，必须用绝对路径或 bash "C:\\path\\script.sh"
    - 案例: 本次修复 settings.json hooks 配置，commit b6aa1f3

Step 2a 分析:
  Grep "Windows" + "hooks" → 找到 #L031（Windows 路径兼容性）
  重叠度: 中等（#L031 讲路径，本条讲 hooks 具体格式）
  写入策略: 合并到 #L031，追加 hooks 专项案例

Step 2b 展示给用户:
  候选经验 #1
  内容: Windows hooks 命令格式必须用 bash "绝对路径"，不能用 ./
  Before: "command": "./hooks/intent-detector.sh"
  After: "command": "bash \"C:\\\\path\\\\hooks\\\\intent-detector.sh\""
  案例引用: settings.json hooks 配置修复，本次对话
  写入策略: 合并到 #L031
  置信度: [UNVERIFIED]

  [等待用户确认]

Step 3 Skill 评估:
  重复模式: 2 次（未达 3 次阈值）
  流程复杂度: 2 步（未达 5 步阈值）
  结论: 不建议提取 Skill，写入 lessons-learned 即可

Step 4 链式收尾:
  ✅ 复盘完成。建议接着调用 /neat 同步三层知识。
```
