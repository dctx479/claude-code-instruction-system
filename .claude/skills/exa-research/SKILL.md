# Exa Research Skill

> 基于 Exa 搜索引擎的企业与市场研究工具，专注于公司情报、竞争对手分析和市场研究

## 契约定义

### What（输入/输出）

**输入**：
- 研究目标（公司名称、行业、市场问题）
- 研究类型（company/news/tweet/people）
- 结果数量偏好（可选："几个"/"全面"/具体数字）
- 输出格式偏好（JSON/Markdown 报告）

**输出**：
- 结构化的公司/市场情报
- 竞争分析报告（含定位矩阵）
- 市场研究报告（含数据点和趋势）
- 精炼后的 JSON 或 Markdown 格式结果

### When Done（验收标准）

- 所有搜索在隔离的 Task Agent 中执行
- 返回结果已精炼（非原始搜索结果）
- 公司信息包含：员工数、位置、融资、营收（如可用）
- 竞争分析包含至少 3 家竞品的对比
- 市场研究包含至少 5 个数据点
- 结果来源可追溯

### What NOT（边界约束）

- **禁止直接使用 web_search**：必须使用 `web_search_advanced`（Exa MCP）
- **禁止主对话执行搜索**：所有搜索必须通过 Task Agent 隔离
- **禁止返回原始结果**：必须精炼处理后返回
- **禁止编造数据**：所有数字必须来自搜索结果
- **禁止单一来源**：重要数据需多查询交叉验证

## 核心架构

### 工具限制

**唯一允许的工具**: `web_search_advanced`（来自 Exa MCP）

```
主对话上下文
    │
    └─→ Task Agent (独立上下文)
            │
            ├─ web_search_advanced(query, category, num_results)
            │       ↓
            ├─ 精炼处理：提取关键信息
            │       ↓
            └─ 返回结构化结果 (JSON/Markdown)
```

### Token 隔离模式

**关键原则**：防止搜索结果污染主对话上下文

```
主对话
    │
    ├─→ Task Agent 1: 公司主页搜索
    │       └─ 返回: {company, employees, location, funding}
    │
    ├─→ Task Agent 2: 新闻搜索
    │       └─ 返回: [{title, date, summary}, ...]
    │
    ├─→ Task Agent 3: 社交媒体搜索
    │       └─ 返回: [{platform, content, engagement}, ...]
    │
    └─ 主对话仅接收精炼后的结构化数据
```

## 搜索类别

Exa 支持以下搜索类别，根据研究需求选择：

| 类别 | 用途 | 期望获取的信息 |
|------|------|---------------|
| **company** | 公司主页 | 员工数、位置、融资轮次、营收、业务描述 |
| **news** | 新闻报道 | 最新动态、融资新闻、产品发布、管理层变动 |
| **tweet** | 社交媒体 | 品牌声量、用户反馈、舆情趋势 |
| **people** | LinkedIn 公开资料 | 创始人背景、高管团队、人才流动 |

### 类别选择策略

```
研究需求分析
    │
    ├─ 公司基本面 → category: company
    │
    ├─ 最新动态 → category: news
    │
    ├─ 舆情分析 → category: tweet
    │
    └─ 团队背景 → category: people
```

## 搜索调优策略

### 结果数量调优

根据用户意图动态调整 `num_results`：

| 用户表述 | 推荐数量 | 适用场景 |
|---------|---------|---------|
| "几个"、"简单看看" | 10-20 | 快速了解 |
| "全面"、"详细" | 50-100 | 深度研究 |
| 具体数字（如"30个"） | 遵循用户 | 明确需求 |
| 未指定 | 询问澄清 | 避免浪费 |

**交互示例**：
> "你希望获得多少条结果？
> - 快速了解（10-20条）
> - 全面分析（50-100条）
> - 其他数量（请指定）"

### 查询优化策略

**并行多查询**：为提高覆盖率，对同一目标运行 2-3 个查询变体

```
目标: 分析 Stripe 的竞争对手

查询 1: "Stripe competitors payment processing"
查询 2: "payment gateway alternatives to Stripe"
查询 3: "fintech companies payment infrastructure"

→ 合并去重 → 精炼输出
```

**查询变体生成原则**：
1. 直接查询（品牌名 + 关键词）
2. 替代查询（同义词、行业术语）
3. 问题式查询（"什么是...的替代品"）

## 降级策略

当 Exa 搜索结果不足时，自动升级到更强能力：

```
Exa 搜索
    ↓
[结果充足?]
    ├─ 是 → 精炼返回
    └─ 否 → [需要什么?]
              │
              ├─ 动态 JavaScript 页面
              │     → 升级到 Playwright MCP
              │
              ├─ 需要认证的内容
              │     → 提示用户手动获取
              │
              └─ 数据过时
                    → 调整时间范围重试
```

**降级触发条件**：
- 返回结果 < 5 条
- 关键字段缺失（如员工数、融资）
- 内容明显过时（> 6 个月）

## 模型选择

根据任务复杂度自动选择模型：

| 任务类型 | 推荐模型 | 理由 |
|---------|---------|------|
| 快速提取 | **Haiku** | 简单信息抽取，成本低 |
| 信息发现 | **Haiku** | 批量搜索，效率优先 |
| 复杂综合 | **Opus** | 多来源整合，需要推理 |
| 浏览器自动化 | **Opus** | 动态页面处理，复杂交互 |

**自动选择逻辑**：
```
任务分析
    │
    ├─ 简单查询（单类别、<20条） → Haiku
    │
    ├─ 标准研究（多类别、20-50条） → Sonnet
    │
    └─ 深度分析（全类别、>50条） → Opus
```

## 工作流程

### 标准研究流程

```
1. 需求理解
   ├─ 确认研究目标
   ├─ 确定搜索类别
   └─ 协商结果数量

2. 搜索执行 (Task Agent)
   ├─ 生成查询变体
   ├─ 并行执行搜索
   └─ 收集原始结果

3. 结果精炼 (Task Agent)
   ├─ 提取关键信息
   ├─ 结构化处理
   └─ 去重和验证

4. 报告生成 (主对话)
   ├─ 整合所有精炼结果
   ├─ 套用报告模板
   └─ 添加分析和建议

5. 交付
   └─ 返回 JSON 或 Markdown 报告
```

### 快速研究流程（简化版）

```
1. 单类别搜索 → 2. 快速精炼 → 3. 返回 JSON
```

## MCP 集成配置

### Exa MCP 配置

在 `config/mcp-servers.json` 中添加：

```json
{
  "exa": {
    "command": "npx",
    "args": ["@anthropic/exa-mcp"],
    "env": {
      "EXA_API_KEY": "${EXA_API_KEY}"
    }
  }
}
```

### 环境变量

在 `~/.claude/settings.json` 中配置：

```json
{
  "env": {
    "EXA_API_KEY": "your-exa-api-key"
  }
}
```

### 获取 API Key

1. 访问 [Exa.ai](https://exa.ai/)
2. 注册并获取 API Key
3. 配置到环境变量

## 使用示例

### 示例 1：公司情报收集

**用户**：帮我调研一下 Notion 这家公司

**执行流程**：
1. 确认研究范围（基本信息 + 融资 + 最新动态）
2. Task Agent 执行：
   - `web_search_advanced("Notion company", category="company")`
   - `web_search_advanced("Notion funding news", category="news")`
3. 精炼返回公司情报 JSON
4. 生成结构化报告

### 示例 2：竞品分析

**用户**：分析一下 Figma 的主要竞争对手

**执行流程**：
1. 确认分析维度（功能、定价、定位）
2. Task Agent 执行：
   - `web_search_advanced("Figma competitors design tools")`
   - `web_search_advanced("design software alternatives Figma")`
   - `web_search_advanced("UI design tools comparison")`
3. 精炼提取竞品列表
4. 对每个竞品执行详细搜索
5. 生成竞争分析报告（含定位矩阵）

### 示例 3：市场研究

**用户**：研究一下中国 SaaS 市场的现状

**执行流程**：
1. 确认研究深度（全面分析，50-100 条）
2. Task Agent 并行执行：
   - 市场规模搜索
   - 主要玩家搜索
   - 行业趋势搜索
   - 投融资动态搜索
3. 合并精炼所有结果
4. 生成市场研究报告

## 限制与注意事项

### 当前限制

1. **API 配额**：Exa 有请求配额限制，大规模研究需注意
2. **实时性**：搜索结果有索引延迟，非实时数据
3. **地域覆盖**：英文内容覆盖更全面，中文内容可能有限
4. **深度限制**：部分深度信息（如内部数据）无法获取

### 最佳实践

1. **明确范围**：研究前明确目标和边界
2. **多角度验证**：重要数据用多查询交叉验证
3. **标注来源**：所有数据标注来源便于追溯
4. **定期更新**：市场变化快，研究结果有时效性
5. **人工复核**：重要决策前人工验证关键数据
