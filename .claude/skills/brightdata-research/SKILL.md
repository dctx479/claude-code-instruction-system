# Bright Data Research Skill

> 电商平台深度调研技能，专注于市场情报收集与竞品分析

## 契约定义

### What（输入/输出）

**输入**：
- 调研主题或产品类别（自然语言描述）
- 目标电商平台（Amazon, eBay, Etsy, Walmart, AliExpress 等）
- 可选：研究模式选择（Quick/Standard/Deep）
- 可选：输出格式偏好（Report/JSON/Markdown）
- 可选：特定关注维度（定价/竞品/趋势/评价等）

**输出**：
- 结构化电商调研报告
- 包含具体数据点（价格、销量、评分、排名）
- 市场洞察和竞争分析
- 保存为 `.md` 或 `.json` 文件（默认路径：`reports/ecommerce-{topic}-{date}.md`）

### When Done（验收标准）

- 报告包含至少 5 个具体数据点（价格/销量/评分等）
- 覆盖目标平台的核心产品数据
- 数据来源可追溯（平台、时间、URL）
- 报告结构清晰（摘要 -> 数据 -> 分析 -> 建议）
- 遵循道德爬虫规范（robots.txt、请求频率）
- 文件已保存到指定路径

### What NOT（边界约束）

- **禁止违反 robots.txt**：必须遵循目标网站的爬虫规则
- **禁止过度请求**：控制请求频率，避免对目标服务器造成压力
- **禁止编造数据**：所有数字必须来自实际抓取结果
- **禁止忽略大页面限制**：Amazon/Etsy 等大页面每批次 1-2 URLs
- **禁止单次处理过多 URLs**：避免 Token 超限（Quick <=5, Standard <=20）
- **禁止私有信息抓取**：不收集用户个人信息
- **必须注明数据来源**：所有数据需标注平台和抓取时间

## 三种研究模式

### Quick Mode（快速模式）

**特点**：
- URLs 数量：3-5 个
- 处理方式：顺序处理
- 适用场景：快速概览、初步探索
- 耗时：1-3 分钟

**使用场景**：
```markdown
# 快速了解某品类概况
"快速看一下 Amazon 上蓝牙耳机的价格区间"

# 单一产品信息
"帮我查一下这个 ASIN 的产品详情"
```

**执行流程**：
```
用户请求 -> 解析目标 -> 搜索发现 (1-2 queries)
    -> 顺序抓取 (3-5 URLs) -> 数据提取 -> 快速报告
```

### Standard Mode（标准模式）

**特点**：
- URLs 数量：10-20 个
- 处理方式：并行处理
- 适用场景：市场分析、竞品研究
- 耗时：5-10 分钟

**使用场景**：
```markdown
# 品类市场分析
"分析 Amazon 上智能手表市场的前 20 名产品"

# 竞品对比
"对比 Apple Watch、Samsung Galaxy Watch、Garmin 在各平台的表现"
```

**执行流程**：
```
用户请求 -> 解析目标 -> 搜索发现 (3-5 queries)
    -> 批量抓取 (10-20 URLs, 并行)
    -> 结构化提取 -> 数据整合 -> 详细报告
```

**注意事项**：
- 大页面（Amazon, Etsy）每批 1-2 URLs
- 小页面可每批 5-10 URLs
- 监控 Token 消耗，避免超限

### Deep Mode（深度模式）

**特点**：
- 使用浏览器自动化
- 支持 JavaScript 渲染内容
- 处理复杂交互场景
- 适用场景：动态内容、需要登录/滚动的页面
- 耗时：10-30 分钟

**使用场景**：
```markdown
# 需要滚动加载的页面
"分析 Instagram Shop 上的热门电商品牌"

# 需要交互的内容
"获取商品的详细规格（需要展开查看）"

# 动态加载的评价
"收集产品的全部用户评价（分页加载）"
```

**执行流程**：
```
用户请求 -> 解析目标 -> 启动浏览器自动化
    -> 页面导航 -> 等待渲染 -> 交互操作（滚动/点击）
    -> 内容提取 -> 数据结构化 -> 深度报告
```

**技术要求**：
- 需要配置 Playwright MCP 或 Bright Data Browser Automation
- 处理速度较慢，但数据更完整

## Bright Data MCP 配置

### 前置要求

1. **Bright Data 账户**：[注册地址](https://brightdata.com/)
2. **API Token**：在控制台获取 API 密钥
3. **MCP 服务器配置**：添加到 `config/mcp-servers.json`

### MCP 配置示例

```json
{
  "mcpServers": {
    "brightdata": {
      "command": "npx",
      "args": ["-y", "@anthropic-ai/brightdata-mcp"],
      "env": {
        "BRIGHTDATA_API_TOKEN": "your-api-token",
        "BRIGHTDATA_ZONE": "your-zone-id"
      }
    }
  }
}
```

### 可用工具

| 工具 | 用途 | 模式支持 |
|------|------|---------|
| **search_discovery** | Google/Bing/Yandex 搜索 | Quick/Standard/Deep |
| **content_scraping** | Markdown/HTML 格式抓取 | Quick/Standard |
| **batch_processing** | 并发 URL 收集 | Standard/Deep |
| **structured_extraction** | AI 驱动的数据识别 | Standard/Deep |
| **browser_automation** | 交互式内容处理 | Deep |

### 代理配置（可选）

```json
{
  "proxy": {
    "type": "residential",
    "country": "US",
    "session_type": "rotating"
  }
}
```

## 电商调研维度

### 1. 市场规模分析

**数据点**：
- 品类总产品数量
- 价格区间分布
- 月销量估算
- 市场增长趋势

**输出示例**：
```markdown
## 市场规模

| 指标 | 数值 | 说明 |
|------|------|------|
| 总产品数 | 15,000+ | 搜索结果页显示 |
| 价格区间 | $15 - $300 | 主流价格带 $50-$100 |
| 月均销量 | 50,000 件 | Top 100 产品估算 |
| YoY 增长 | +25% | 基于历史数据对比 |
```

### 2. 畅销排行分析

**数据点**：
- Top 10/20/50 产品列表
- 销量排名
- 评分和评价数
- 价格定位

**输出示例**：
```markdown
## 畅销排行 Top 10

| 排名 | 产品名称 | 品牌 | 价格 | 评分 | 评价数 | 月销量估算 |
|------|---------|------|------|------|--------|-----------|
| 1 | XXX | BrandA | $79.99 | 4.7 | 25,000 | 5,000 |
| 2 | YYY | BrandB | $59.99 | 4.5 | 18,000 | 4,200 |
| ... | ... | ... | ... | ... | ... | ... |
```

### 3. 品牌份额分析

**数据点**：
- Top 品牌列表
- 市场份额占比
- 品牌定位（高端/中端/低端）
- 品牌产品线覆盖

**输出示例**：
```markdown
## 品牌市场份额

| 品牌 | 市场份额 | 产品数量 | 平均评分 | 定位 |
|------|---------|---------|---------|------|
| BrandA | 25% | 150 | 4.6 | 高端 |
| BrandB | 18% | 200 | 4.4 | 中端 |
| BrandC | 12% | 300 | 4.2 | 性价比 |
```

### 4. 定价策略分析

**数据点**：
- 价格分布直方图
- 价格带划分
- 促销频率
- 价格变动趋势

**输出示例**：
```markdown
## 定价策略

### 价格带分布
- 低端 ($0-30): 30%
- 中端 ($30-80): 45%
- 高端 ($80-150): 20%
- 奢侈 ($150+): 5%

### 定价建议
- 主流价格带：$50-$80
- 竞争激烈区间：$30-$50
- 蓝海机会：$100-$120（高端但未饱和）
```

### 5. 竞争格局分析

**数据点**：
- 主要竞争者
- 差异化特点
- 竞争优劣势
- 市场进入壁垒

### 6. 技术趋势分析

**数据点**：
- 热门功能/特性
- 新技术应用
- 消费者偏好变化
- 创新方向

## 使用示例

### 示例 1: 快速市场概览（Quick Mode）

**用户**: 快速看一下 Amazon 上无线耳机的市场情况

**执行流程**:
1. 识别为 Quick Mode（快速概览）
2. 使用 search_discovery 搜索 "wireless earbuds site:amazon.com"
3. 顺序抓取 Top 5 搜索结果
4. 提取价格、评分、销量数据
5. 生成快速概览报告

**输出示例**:
```markdown
## 无线耳机市场快速概览

### 核心发现
- 主流价格带：$30-$80
- Top 品牌：Apple、Samsung、Sony、JBL、Anker
- 平均评分：4.3/5.0

### Top 5 产品
1. Apple AirPods Pro - $249 - 4.7星 - 100K+ 评价
2. Samsung Galaxy Buds - $149 - 4.5星 - 50K+ 评价
...
```

### 示例 2: 竞品深度分析（Standard Mode）

**用户**: 分析 Amazon 上智能手表市场，对比 Apple、Samsung、Garmin 三个品牌

**执行流程**:
1. 识别为 Standard Mode（竞品分析）
2. 搜索发现三个品牌的产品列表页
3. 并行抓取 15-20 个产品页面
4. 结构化提取产品数据
5. 对比分析三个品牌
6. 生成详细竞品报告

### 示例 3: 跨平台价格监控（Deep Mode）

**用户**: 深度分析某款产品在 Amazon、eBay、Walmart 的价格差异和评价

**执行流程**:
1. 识别为 Deep Mode（跨平台深度分析）
2. 启动浏览器自动化
3. 分别访问三个平台的产品页
4. 处理动态加载内容（评价、规格等）
5. 跨平台数据对比
6. 生成深度分析报告

## 道德规范与合规

### 必须遵守

1. **robots.txt 合规**
   - 检查并遵循目标网站的 robots.txt
   - 尊重 Crawl-delay 指令
   - 不访问禁止路径

2. **请求频率控制**
   - 单域名请求间隔 >= 2 秒
   - 避免并发过高导致服务器压力
   - 使用 Bright Data 的智能节流

3. **数据使用规范**
   - 仅收集公开可见信息
   - 不收集用户个人信息
   - 正确引用数据来源
   - 不用于违法目的

4. **版权尊重**
   - 不大规模复制受版权保护的内容
   - 合理使用原则
   - 标注数据来源

### 风险提示

- 部分平台可能有反爬虫机制，数据获取可能不完整
- 销量数据通常为估算值，仅供参考
- 价格数据可能因地区、时间、促销等因素变化
- 建议定期更新数据，避免使用过时信息

## 性能注意事项

### Token 管理

| 页面类型 | 每批 URLs 上限 | 原因 |
|----------|---------------|------|
| Amazon 产品页 | 1-2 | 页面内容多，Token 消耗高 |
| Etsy 产品页 | 1-2 | 同上 |
| eBay 列表页 | 3-5 | 内容中等 |
| 一般小页面 | 5-10 | 内容较少 |

### 错误处理

```markdown
遇到 Token 超限时：
1. 减少单批 URL 数量
2. 切换到 Quick Mode
3. 分批处理后合并结果

遇到抓取失败时：
1. 检查 URL 有效性
2. 尝试使用代理轮换
3. 切换到 Deep Mode（浏览器模式）
```

### 成本优化

- Quick Mode：成本最低，适合初步探索
- Standard Mode：性价比最高，适合常规调研
- Deep Mode：成本较高，仅在必要时使用

## 限制与注意事项

### 当前限制

1. **平台覆盖**：主要支持英文电商平台，中文平台支持有限
2. **实时性**：数据为抓取时刻的快照，非实时
3. **准确性**：销量为估算值，实际数据以平台为准
4. **频率限制**：受 Bright Data API 配额限制

### 最佳实践

1. **明确调研目标**：提供清晰的品类、品牌、平台范围
2. **选择合适模式**：根据需求选择 Quick/Standard/Deep
3. **分批处理大任务**：避免单次处理过多 URLs
4. **交叉验证**：重要数据建议多平台验证
5. **定期更新**：电商数据变化快，定期刷新
