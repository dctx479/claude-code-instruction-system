# Social Media Research Skill

> 跨平台社交媒体数据研究技能，专注于品牌舆情监控、KOL 分析和内容趋势洞察

## 契约定义

### What（输入/输出）

**输入**：
- 研究主题或品牌/产品名称（自然语言描述）
- 目标平台（可多选）：TikTok, Douyin, Xiaohongshu, Bilibili, Kuaishou, Weibo, Instagram, YouTube, Twitter, Threads, Reddit, Zhihu 等
- 可选：研究类型（舆情监控/KOL 分析/内容趋势/竞品分析）
- 可选：时间范围（最近 7 天/30 天/自定义）
- 可选：输出格式（Report/Dashboard/JSON）

**输出**：
- 结构化社媒研究报告
- 包含具体数据点（互动量、粉丝数、增长率、情感倾向）
- 可视化数据洞察（趋势图表、词云、热度排名）
- 保存为 `.md` 文件（默认路径：`reports/social-media-{topic}-{date}.md`）

### When Done（验收标准）

- 报告包含至少 10 个具体数据点（互动量/粉丝数/发布量等）
- 覆盖至少 1 个目标平台的核心数据
- 数据来源可追溯（平台、账号、时间）
- 报告结构清晰（摘要 -> 数据 -> 分析 -> 建议）
- 情感分析结果（如适用）包含正面/中性/负面比例
- 文件已保存到指定路径

### What NOT（边界约束）

- **禁止编造数据**：所有数字必须来自 API 返回结果
- **禁止过度请求**：遵循 10 请求/秒/端点的限流规则
- **禁止私有信息采集**：不收集用户私信、隐私数据
- **禁止单次处理过多**：每次 API 调用控制在合理范围
- **禁止跳过错误处理**：API 失败时必须有重试和降级策略
- **禁止忽略时效性**：报告需注明数据采集时间
- **必须注明数据来源**：所有数据需标注平台和采集时间

## 核心架构

### TikHub API 架构

```
┌─────────────────────────────────────────────────────────────┐
│                    TikHub MCP Server                         │
│  - API 认证管理                                              │
│  - 请求限流控制 (10 req/s/endpoint)                         │
│  - 自动重试 (最多 3 次)                                      │
└───────────────────────┬─────────────────────────────────────┘
                        │
        ┌───────────────┼───────────────┬───────────────┐
        │               │               │               │
        ▼               ▼               ▼               ▼
┌──────────────┐ ┌──────────────┐ ┌──────────────┐ ┌──────────────┐
│  API Searcher │ │  API Client  │ │   OpenAPI    │ │  Batch Job   │
│  关键词发现   │ │  直接调用    │ │   规范文档   │ │  批量任务    │
│  端点         │ │  (内置认证)  │ │              │ │              │
└──────────────┘ └──────────────┘ └──────────────┘ └──────────────┘
```

### API 端点格式

```
/api/v1/{platform}/{method}/{action}
```

**示例端点**：
- `/api/v1/tiktok/web/search_video` - TikTok 视频搜索
- `/api/v1/douyin/web/fetch_user_profile` - 抖音用户资料
- `/api/v1/xiaohongshu/web/search_notes` - 小红书笔记搜索
- `/api/v1/bilibili/web/fetch_video_info` - B站视频信息

## TikHub MCP 配置

### 前置要求

1. **TikHub 账户**：[注册地址](https://www.tikhub.io/)
2. **API Token**：在控制台获取 API 密钥
3. **MCP 服务器配置**：添加到 `config/mcp-servers.json`

### MCP 配置示例

```json
{
  "mcpServers": {
    "tikhub": {
      "command": "npx",
      "args": ["-y", "@anthropic-ai/tikhub-mcp"],
      "env": {
        "TIKHUB_API_TOKEN": "your-api-token"
      }
    }
  }
}
```

### 自定义 Token 配置（可选）

如果用户有自己的 TikHub 账户：

```json
{
  "mcpServers": {
    "tikhub": {
      "command": "npx",
      "args": ["-y", "@anthropic-ai/tikhub-mcp"],
      "env": {
        "TIKHUB_API_TOKEN": "${TIKHUB_USER_TOKEN}"
      }
    }
  }
}
```

### 可用工具

## 四种研究模式

### 1. 舆情监控模式

**目标**：监控品牌/产品在社媒上的舆论走向

**数据维度**：
- 声量趋势（提及次数随时间变化）
- 情感分析（正面/中性/负面比例）
- 关键词云（高频提及词）
- 负面预警（识别潜在危机）

**执行流程**：
```
品牌关键词 -> 多平台搜索 -> 内容采集
    -> 情感分析 -> 趋势计算 -> 预警识别 -> 舆情报告
```

### 2. KOL 分析模式

**目标**：识别和评估领域内有影响力的创作者

**数据维度**：
- 粉丝量和增长趋势
- 互动率（点赞/评论/分享比率）
- 内容质量评分
- 粉丝画像匹配度
- 商业合作价值评估

**执行流程**：
```
领域关键词 -> 搜索相关内容 -> 提取创作者列表
    -> 逐一获取创作者详情 -> 计算评估指标 -> KOL 报告
```

### 3. 内容趋势模式

**目标**：发现平台上的热门内容和趋势话题

**数据维度**：
- 热门话题/标签
- 爆款内容特征
- 发布时间规律
- 内容形式偏好
- 互动数据基准

**执行流程**：
```
品类/话题 -> 热门内容搜索 -> 特征提取
    -> 趋势分析 -> 规律总结 -> 内容策略建议
```

### 4. 竞品分析模式

**目标**：分析竞品在社媒上的表现和策略

**数据维度**：
- 竞品账号矩阵
- 内容发布频率
- 互动数据对比
- 爆款内容分析
- 营销策略推断

**执行流程**：
```
竞品品牌列表 -> 搜索官方/相关账号 -> 数据采集
    -> 横向对比 -> 策略分析 -> 竞品报告
```

## 限流与错误处理

### 限流规则

| 限制类型 | 限制值 | 处理策略 |
|----------|-------|---------|
| 请求频率 | 10 请求/秒/端点 | 自动节流 |
| 单次超时 | 30-60 秒 | 超时重试 |
| 重试次数 | 最多 3 次 | 指数退避 |
| 批量任务 | 根据账户等级 | 分批提交 |

### 错误处理策略

```markdown
API 调用
    ↓
[成功?]
├─ 是 → 处理数据
└─ 否 → [错误类型?]
         ├─ 429 (限流) → 等待后重试
         ├─ 401 (认证) → 检查 Token
         ├─ 404 (不存在) → 记录并跳过
         ├─ 500 (服务器) → 重试或降级
         └─ 其他 → 记录错误，继续处理其他数据
```

### 最佳实践

1. **分批处理**：大规模数据采集时分批进行
2. **优先级队列**：重要数据优先采集
3. **缓存机制**：避免重复请求相同数据
4. **降级策略**：部分平台失败不影响整体报告

## 使用示例

### 示例 1: 品牌舆情监控

**用户**: 帮我监控"完美日记"在小红书和抖音上最近一周的舆情

**执行流程**:
1. 识别为舆情监控模式
2. 调用小红书和抖音搜索 API
3. 采集关键词相关内容（品牌名、产品名、活动等）
4. 执行情感分析（正面/中性/负面分类）
5. 计算声量趋势和热度
6. 识别风险点和负面舆论
7. 生成舆情监控报告

**API 调用示例**:
```
1. search_api("小红书 笔记搜索") -> 获取端点
2. call_api("/api/v1/xiaohongshu/web/search_notes", {"keyword": "完美日记"})
3. call_api("/api/v1/douyin/web/search_video", {"keyword": "完美日记"})
4. 循环获取内容详情和评论
5. 情感分析处理
```

### 示例 2: KOL 筛选评估

**用户**: 帮我找出抖音上美妆领域粉丝量 10-50 万、互动率高的 KOL

**执行流程**:
1. 识别为 KOL 分析模式
2. 搜索美妆相关热门内容
3. 提取创作者列表
4. 逐一获取创作者详情（粉丝数、作品数据）
5. 计算互动率（点赞+评论+分享/粉丝数）
6. 筛选符合条件的 KOL
7. 排序并生成推荐列表

**API 调用示例**:
```
1. call_api("/api/v1/douyin/web/search_video", {"keyword": "美妆教程"})
2. 提取 author_id 列表
3. call_api("/api/v1/douyin/web/fetch_user_profile", {"user_id": "xxx"})
4. call_api("/api/v1/douyin/web/fetch_user_post", {"user_id": "xxx"})
5. 计算互动率并排序
```

### 示例 3: 内容趋势分析

## 限制与注意事项

### 当前限制

1. **平台覆盖**：部分平台 API 能力有限（如微信）
2. **数据实时性**：数据为 API 调用时刻的快照
3. **情感分析**：自动情感分析可能存在误判
4. **历史数据**：部分平台无法获取历史数据
5. **私有数据**：无法获取平台不公开的数据（如精准转化数据）

### 最佳实践

1. **明确研究目标**：提供清晰的品牌、平台、时间范围
2. **选择合适平台**：根据目标用户群体选择重点平台
3. **分批处理大任务**：避免单次 API 调用过多
4. **交叉验证**：重要数据建议多平台验证
5. **定期更新**：社媒数据变化快，定期刷新
6. **人工复核**：情感分析结果建议人工抽检

### 道德规范

- 仅收集公开可见信息
- 不追踪或收集用户私人信息
- 正确引用数据来源
- 不用于恶意竞争或违法目的
- 尊重平台使用条款
