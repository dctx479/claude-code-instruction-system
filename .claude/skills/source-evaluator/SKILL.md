# Source Evaluator Skill（伯乐）

> "学习材料的质量，决定了你的学习效率。垃圾进，垃圾出。"

## 契约定义

### What（输入/输出）

**输入**：
- 学习材料描述（文章链接/书籍名称/课程名称/代码库）
- 材料内容摘要或关键观点（可选）
- 评估场景：个人学习/团队分享/知识库收录（可选）

**输出**：
- 五维度评估报告（每维度 1-10 分 + 理由）
- 总评分（加权计算，满分 100）
- 学习建议：读/不读/有条件阅读
- 知识库收录建议：收录/不收录/提取精华后收录

### When Done（验收标准）

- [ ] 五个维度均有明确评分（1-10）
- [ ] 每个维度都有具体评分理由（非泛泛而谈）
- [ ] 计算了加权总分（满分 100）
- [ ] 给出了明确的学习建议
- [ ] 如果是知识库收录场景，给出了收录建议

### What NOT（边界约束）

- ❌ **禁止凭印象评分** — 必须基于材料实际内容
- ❌ **禁止权威崇拜** — 大V/知名机构的内容也要客观评估
- ❌ **禁止一刀切** — 不同场景（自学/教学/研究）评估重点不同
- ❌ **禁止只看标题** — 必须看实际内容，标题党会误导
- ❌ **禁止没有建议** — 评估后必须给出可操作的建议

## 核心能力

### 五维度评估框架

| 维度 | 权重 | 评估要点 | 高分特征 | 低分特征 |
|------|------|----------|----------|----------|
| **深度** | 30% | 是否触及本质/是否有独特洞察 | 揭示原理、有实证数据、可验证 | 浅尝辄止、流于表面、概念堆砌 |
| **准确性** | 25% | 事实是否正确/逻辑是否严密 | 引用权威、可追溯、逻辑自洽 | 错误百出、逻辑混乱、无据可查 |
| **可操作性** | 20% | 是否可直接应用/有无具体方法 | 有步骤、有案例、可复现 | 纯理论、无实例、空谈概念 |
| **新颖性** | 15% | 是否有新观点/是否超越常识 | 颠覆认知、少数派观点、前沿 | 陈词滥调、拾人牙慧、过时 |
| **表达质量** | 10% | 是否易懂/结构是否清晰 | 逻辑清晰、图文并茂、易读 | 晦涩难懂、结构混乱、冗长 |

### 评分标准

| 总分 | 评级 | 学习建议 | 知识库收录 |
|------|------|----------|-----------|
| 85-100 | ⭐⭐⭐⭐⭐ 珍宝 | 精读，反复研究，沉淀笔记 | 全文收录 + 标注重点 |
| 70-84 | ⭐⭐⭐⭐ 优质 | 认真学习，可作为参考 | 收录精华部分 |
| 55-69 | ⭐⭐⭐ 一般 | 选择性阅读，提取有用部分 | 仅收录核心观点 |
| 40-54 | ⭐⭐ 低质 | 快速浏览，不值得深入 | 不收录 |
| <40 | ⭐ 垃圾 | 不建议阅读，浪费时间 | 不收录 |

## How（执行步骤）

### 阶段 1：材料扫描

**快速获取材料关键信息**：

```markdown
1. 识别材料类型：文章/书籍/课程/代码/论文
2. 提取核心观点（3-5 条）
3. 识别目标受众（初学者/中级/专家）
4. 检查信息源：作者背景、发布平台、发布时间
```

### 阶段 2：五维度打分

**逐维度评估，给出具体理由**：

```markdown
## 五维度评估

### 1. 深度（权重 30%）得分：X/10

**评分理由**：
- [具体例子说明深度]
- [是否触及本质]

### 2. 准确性（权重 25%）得分：X/10

**评分理由**：
- [事实核查结果]
- [逻辑推理是否严密]

### 3. 可操作性（权重 20%）得分：X/10

**评分理由**：
- [是否有具体方法]
- [能否直接应用]

### 4. 新颖性（权重 15%）得分：X/10

**评分理由**：
- [是否有独特观点]
- [是否颠覆常识]

### 5. 表达质量（权重 10%）得分：X/10

**评分理由**：
- [可读性如何]
- [结构是否清晰]
```

### 阶段 3：综合判断

**计算总分并给出建议**：

```markdown
## 综合评估

**总评分**：XX/100
- 深度：X/10 × 30% = Y
- 准确性：X/10 × 25% = Y
- 可操作性：X/10 × 20% = Y
- 新颖性：X/10 × 15% = Y
- 表达质量：X/10 × 10% = Y

**评级**：⭐⭐⭐⭐⭐ 珍宝 / ⭐⭐⭐⭐ 优质 / ⭐⭐⭐ 一般 / ⭐⭐ 低质 / ⭐ 垃圾

**学习建议**：
- [读/不读/有条件阅读]
- [如何阅读：精读/泛读/跳读]
- [重点关注哪些章节]

**知识库收录建议**：
- [收录/不收录/提取精华]
- [如果收录，收录哪些部分]
- [需要补充哪些上下文]
```

## 使用示例

### 示例 1：评估技术文章

**用户输入**：
> "评估这篇文章的学习价值：《我给 AI 装了 80 多个 Skill，最后只保留了这一个》"

**评估报告**：

```markdown
## 五维度评估

### 1. 深度（30%）得分：9/10
**评分理由**：
- 揭示了 Skill 系统的本质问题：Token 消耗 vs 能力扩展
- 提出了路由器设计模式，有理论支撑（渐进式披露）
- 从实践中总结（80 个 Skill 的真实经验）

### 2. 准确性（25%）得分：8/10
**评分理由**：
- 提出的 Token 节省数据（95%）可验证
- 路由决策表基于实际场景，逻辑自洽
- 有一处可商榷：负向路由的必要性（可能过度设计）

### 3. 可操作性（20%）得分：9/10
**评分理由**：
- 提供了完整的实现路径（intent-state.json + INDEX.md）
- 有具体的代码示例和配置方式
- 可直接应用到现有系统

### 4. 新颖性（15%）得分：8/10
**评分理由**：
- 路由器模式不新，但应用到 Skill 管理是创新
- 负向路由（告诉 AI 不需要什么）是新颖视角
- 与现有 Skill 市场思路不同

### 5. 表达质量（10%）得分：8/10
**评分理由**：
- 逻辑清晰，从问题到方案层层递进
- 有对比表格和示例代码
- 部分技术细节略显冗长

## 综合评估

**总评分**：85/100
- 深度：9/10 × 30% = 27
- 准确性：8/10 × 25% = 20
- 可操作性：9/10 × 20% = 18
- 新颖性：8/10 × 15% = 12
- 表达质量：8/10 × 10% = 8

**评级**：⭐⭐⭐⭐⭐ 珍宝

**学习建议**：
- **精读推荐**：路由器设计模式、渐进式披露机制、负向路由原则
- **可跳过**：具体代码实现细节（根据自己技术栈调整）
- **建议实践**：先从 10 个 Skill 开始验证，再扩展到更多

**知识库收录建议**：
- **收录策略**：全文收录 + 重点标注
- **重点部分**：路由决策表、三大职责、五大原则
- **补充上下文**：建议关联 `docs/ORCHESTRATION-GUIDE.md` 中的编排模式
```

### 示例 2：评估代码库

**用户输入**：
> "评估学习价值：GitHub 上某个 React 组件库"

**评估报告**（简化版）：

```markdown
## 综合评估

**总评分**：62/100
- 深度：5/10（只是组件封装，无架构设计洞察）
- 准确性：7/10（代码质量一般，有些边界情况未处理）
- 可操作性：8/10（文档完整，可直接使用）
- 新颖性：4/10（常见组件，无创新）
- 表达质量：7/10（文档清晰，但示例不够丰富）

**评级**：⭐⭐⭐ 一般

**学习建议**：
- **适合场景**：快速开发项目，需要现成组件
- **不适合场景**：想学习 React 架构设计、组件设计原则
- **阅读重点**：API 文档和使用示例，源码可选读

**知识库收录建议**：
- **不收录**：已有类似组件库（Ant Design/Material-UI）
- **如需收录**：仅收录 API 文档和使用注意事项
```

## 系统集成

### 与其他 Skills 的协作

| Skill | 协作方式 | 典型场景 |
|-------|----------|----------|
| **reflection** | 评估后决定是否沉淀经验 | 复盘时评估学到的知识质量 |
| **neat** | 知识库整理时过滤低质内容 | 定期审计知识库，清理垃圾 |
| **literature-mentor** | 论文阅读前评估价值 | 筛选高质量论文，避免时间浪费 |
| **deep-research** | 调研前评估信息源质量 | 确保调研材料可靠 |

### 典型工作流

```
用户收集学习材料
    ↓
[1] source-evaluator 评估质量
    ↓
[2] 高分材料（≥70）→ 深入学习
    中分材料（55-69）→ 选择性阅读
    低分材料（<55）→ 跳过
    ↓
[3] 学习后用 reflection 复盘
    ↓
[4] 优质经验用 neat 整理进知识库
```

## 质量检查清单

评估完成后自动检查：

- [ ] 五个维度是否都有评分
- [ ] 每个维度是否有具体理由（非"质量不错"等模糊表述）
- [ ] 总分计算是否正确
- [ ] 是否给出了明确的学习建议
- [ ] 是否给出了知识库收录建议

## 最佳实践

### 何时使用此 Skill

**适合场景**：
- 面对海量信息，需要快速筛选
- 团队知识库建设，需要质量把关
- 个人学习规划，避免低效学习
- 文献调研前的预筛选

**不适合场景**：
- 已明确必须学习的内容（如公司内部文档、课程作业）
- 娱乐休闲类内容（本 Skill 聚焦学习场景）

### 提高评估准确性的技巧

1. **实际查看内容** — 不要只看标题和简介
2. **对比同类材料** — 横向对比能更准确判断质量
3. **考虑时效性** — 技术类内容注意发布时间
4. **识别目标受众** — 适合初学者的材料对专家可能价值不大

### 避免的误区

- ❌ 权威崇拜 — 知名作者也可能写出低质量内容
- ❌ 完美主义 — 70 分以上的材料就值得学习，不必追求满分
- ❌ 一次性评估 — 学习后可以重新评估，调整评分
