# 股票投资研究扩展

> **设计哲学**: 模拟理性投资者（巴菲特式）的尽调流程 — 大量阅读、多维交叉、多空平衡。
> 基于 8 阶段顺序管道，每个阶段的输出作为下一阶段的输入，形成完整的投资决策链。

## 2. 分析维度（8 阶段顺序管道）

> **关键差异**: 股票分析是**依赖链**而非独立维度。Phase N 的输出喂入 Phase N+1，
> 因此 Lead Agent 应采用 **SEQUENTIAL + PARALLEL 混合**编排：阶段间顺序执行，阶段内多 Subagent 并行。

### 阶段依赖图

```
Phase 0: 执行摘要 ← (所有阶段的综合，最后撰写)
    ↑
Phase 1: 公司事实底座 ──→ Phase 2: 行业周期 ──→ Phase 3: 业务拆解
    │                                                    │
    └──────────────────→ Phase 4: 财务质量 ←─────────────┘
                              │
                         Phase 5: 股权治理
                              │
                         Phase 6: 市场分歧（多空平衡）
                              │
                         Phase 7: 估值护城河
                              │
                         Phase 0: 执行摘要（最终综合）
```

### 各阶段详细定义

| 阶段 | 名称 | 说明 | Subagent 数 | 关键产出 |
|------|------|------|-------------|---------|
| **Phase 1** | 公司事实底座 | 公司基本面：历史沿革、主营业务、管理层、股权结构 | 2-3 | 公司画像卡片 |
| **Phase 2** | 行业周期分析 | 所处行业的周期位置、竞争格局、政策环境、技术趋势 | 2-3 | 行业定位矩阵 |
| **Phase 3** | 业务拆解 | 收入结构、产品线/业务段拆解、增长引擎、客户集中度 | 2-3 | 收入瀑布图数据 |
| **Phase 4** | 财务质量 | 三表分析、盈利质量、现金流健康度、杜邦分解 | 3-4 | 财务健康评分 |
| **Phase 5** | 股权治理 | 股东结构、管理层激励、关联交易、ESG 表现 | 1-2 | 治理风险评级 |
| **Phase 6** | 市场分歧 | 多空双方论点、机构持仓变化、卖空数据、市场情绪 | 3-4 | 多空平衡矩阵 |
| **Phase 7** | 估值护城河 | 多模型估值(DCF/相对/NAV)、护城河评估、安全边际 | 2-3 | 估值区间 + 信号 |

**Subagent 总计**: 15-22 个（快速模式 6-8 个，仅执行 Phase 1+4+7）

### 快速模式 vs 完整模式

| 模式 | 执行阶段 | Subagent 数 | 适用场景 |
|------|---------|-------------|---------|
| **快速概览** | Phase 0 + 1 + 7 | 4-6 | 初步筛选、快速了解 |
| **标准分析** | Phase 0-4 + 7 | 11-16 | 常规投资决策 |
| **完整尽调** | Phase 0-7 | 15-22 | 重仓决策、深度研究 |

### Citation 领域调整

**强制标注规则**：
- 财务数据必须标注**报告期**（如 "2025Q3 季报"）和**数据来源**
- 估值数据标注**计算日期**和**所用假设**（如折现率、增长率）
- 行业排名标注**排名机构**和**统计时间**
- 机构持仓数据标注**披露截止日**（通常滞后 45 天）

**交叉验证重点**：
- 同一财务指标需从至少 2 个独立来源验证（如年报 + Wind）
- 估值结论需至少 2 种方法交叉（如 DCF + 相对估值）
- 行业数据需标注不同来源的口径差异

## 5. 编排策略建议

### Lead Agent 调度方案

```
Round 1 (并行): Phase 1 (3 agents) + Phase 2 (3 agents)
    ↓ 等待完成，Lead Agent 综合公司+行业画像
Round 2 (并行): Phase 3 (3 agents) + Phase 4 (4 agents)
    ↓ 等待完成，Lead Agent 综合业务+财务分析
Round 3 (并行): Phase 5 (2 agents) + Phase 6 (4 agents) + Phase 7 (3 agents)
    ↓ 等待完成，Lead Agent 综合治理+情绪+估值
Final: Lead Agent 撰写 Phase 0 执行摘要（综合所有阶段）
```

- **max_rounds**: 建议设为 3（对应 3 轮并行批次）
- **总 Subagent 数**: 完整模式 19-22 个，快速模式 6-8 个
- **编排模式**: HIERARCHICAL（Lead Agent 主控）+ 轮内 PARALLEL

### 质量门控

每轮结束后 Lead Agent 执行质量检查：

| 检查项 | 不通过时处理 |
|--------|-------------|
| 关键财务数据缺失（营收/利润/现金流） | 补充搜索，不进入下一轮 |
| 行业数据仅单一来源 | 追加 1 个 Subagent 交叉验证 |
| 多空论点不平衡（一方 <2 个论点） | 追加 Subagent 专门搜索弱势方论点 |
| 估值模型仅用 1 种方法 | 追加 Subagent 补充第 2 种估值 |
